# 多方法选举投票平台需求规格说明书 (SRS)

## 1. 项目概述

本项目旨在开发一款全开放、不局限场景的 SaaS 型智能投票平台。系统通过支持多种选举算法（单选、多选、权重、Borda、打分制）、企业级 API 联动及 WebHook 实时推送，满足从互联网趣味评选到严谨企业选举的多样化需求。系统内置高性能防刷引擎与多租户权限体系，确保投票的真实性与合规性。

------

## 2. 角色定义

- **R1 发起人（个人/企业主账号）**：拥有创建、管理投票，调用 API 及管理子账号的最高权限。
- **R2 企业子账号**：由主账号分配权限，可执行投票发布或数据查看，无权删除数据。
- **R3 投票者**：必须注册登录。包含普通个体及通过企业数据库导入的白名单成员。
- **R4 后台管理员**：全平台运营方。负责人工二次审核、申诉处理及系统级熔断干预。
- **R5 第三方系统**：通过 API/WebHook 与本平台进行数据交互的外部应用。

------

## 3. 功能需求 (Functional Requirements)

### 【投票创建与管理模块】

**FR-01：多算法投票创建**

- **功能描述**：发起人可根据需求选择不同的投票算法并配置选项。
- **参与角色**：发起人、企业子账号。
- **前置条件**：账号处于正常状态且未触发熔断。
- **输入**：标题、选项、算法类型（单选/多选/权重/排序/打分）、结束条件。
- **处理流程**：系统渲染动态表单 -> 算法配置 -> 敏感词/图片 OCR 算法初步审核。
- **输出**：生成投票链接、二维码或 H5 嵌入代码。
- **异常情况**：触发敏感词拦截则进入待申诉状态；图片违规直接禁止发布。

**FR-02：白名单设置与邀请**

- **功能描述**：支持通过手机号导入白名单，限定特定人群投票。
- **参与角色**：发起人、企业子账号。
- **输入**：手机号名单列表。
- **处理流程**：系统记录名单 -> 校验参与者登录手机号。
- **输出**：白名单生效标识。
- **异常情况**：若手机号格式错误则提示重新导入。

### 【投票执行模块】

**FR-03：鉴权与投票提交**

- **功能描述**：投票者在有效期内提交选择，系统执行防刷校验。
- **参与角色**：投票者。
- **前置条件**：用户已登录，且通过白名单验证（若有）。
- **输入**：选项选择数据。
- **处理流程**：设备指纹+IP频率校验 -> 人机滑块验证 -> 记录投票。
- **输出**：投票成功提示（或作弊者的“静默成功”）。
- **异常情况**：非白名单用户提示“您不在邀请名单内”并引导注册。

### 【API 与数据交互模块】

**FR-04：API 远程管理接口**

- **功能描述**：支持第三方系统远程创建投票。
- **参与角色**：第三方系统。
- **前置条件**：持有有效的 API Key。
- **处理流程**：校验 Key -> 解析 JSON 指令 -> 系统内自动创建并返回链接。
- **输出**：创建成功的元数据。

**FR-05：WebHook 实时推送**

- **功能描述**：当投票状态变更或有新投票时，实时推送数据至第三方。
- **参与角色**：第三方系统。
- **处理流程**：事件触发 -> 构造 Data Payload -> 发起 POST 请求。
- **异常情况**：推送失败则在 10 分钟后重推，最多重试 3 次。

## 4. 业务规则 (Business Rules)

| **编号**  | **规则名称**           | **规则详细描述**                                             |
| --------- | ---------------------- | ------------------------------------------------------------ |
| **BR-01** | **生命周期与编辑限制** | 审核中、进行中、已结束后禁止修改题目选项。进行中允许“紧急暂停”。归档后前端隐藏但 API 可查。 |
| **BR-02** | **自动结束规则**       | 达到截止时间或达到设定的投票人数上限时，系统自动切换至“已结束”状态。 |
| **BR-03** | **防刷风控策略**       | 强制登录投票。对于异常流量采用“静默丢弃”策略：前端显示成功，后台不计票，并记录设备黑名单。 |
| **BR-04** | **子账号独立规则**     | 主账号封禁时，子账号可选择与主账号解绑并保留账号独立性；子账号无权执行删除数据操作。 |
| **BR-05** | **隐私脱敏规则**       | 系统底层锁死实名名单，发起人仅能查看脱敏后的百分比或统计图，不可追溯个人选票。 |
| **BR-06** | **申诉处理逻辑**       | 算法误杀进入申诉期时，投票维持封禁状态，直到人工审核给出最终结果。 |

------

## 5. 非功能需求 (Non-Functional Requirements)

- **NFR-01 SaaS 熔断保护**：单账号 QPS > 10 时封禁 1 分钟；存储达 80% 发起预警；欠费账号 API 仅返回统计图数据。
- **NFR-02 响应性能**：支持云原生弹性伸缩，确保百万级流量下的 WebHook 推送延迟低于 5 秒。
- **NFR-03 鉴权安全**：Token 有效期为 1 个月，不提供“记住我”本地存储功能。
- **NFR-04 国际化**：系统全链路支持中英双语切换。
